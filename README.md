## Архитектура приложения
Выбран **Вариант Б (Fullstack Next.js)**:
- Фронтенд и бэкенд работают внутри одного приложения Next.js на порту `3000`.
- Серверная логика и CRUD-операции реализованы через Route Handlers в директории `app/api/`.
- Хранилище данных: массивы в оперативной памяти (in-memory) в модуле `lib/store.ts` без подключения сторонних баз данных.

## Технологический стек
- Next.js 14+ (App Router)
- TypeScript
- Tailwind CSS
- Стандартный fetch для сетевых запросов

## Как запустить проект локально
1. Установить зависимости:
   ```bash
   npm install
   ```
2. Запустить сервер разработки:
   ```bash
   npm run dev
   ```
3. Открыть в браузере: `http://localhost:3000`

## Структура данных (Один-ко-многим)

### 1. Рубрика (`Category`)
- `id` (строка, UUID) — уникальный идентификатор.
- `name` (строка) — обязательное название рубрики.
- `description` (строка) — опциональное описание.
- `articlesCount` (число) — количество статей.
- `isArchived` (булево) — статус архива.
- `createdAt` / `updatedAt` (строка) — даты создания и изменения.

### 2. Статья (`Article`)
- `id` (строка, UUID) — уникальный идентификатор.
- `title` (строка) — обязательный заголовок.
- `content` (строка) — обязательный текст.
- `categoryId` (строка) — ID связанной рубрики.
- `views` (число) — количество просмотров.
- `isPublished` (булево) — статус публикации.
- `createdAt` / `updatedAt` (строка) — даты создания и изменения.

## 🔌 Описание API-эндпоинтов

### Рубрики
- `GET /api/categories?page=1&limit=10` — получить список рубрик с пагинацией.
- `POST /api/categories` — создать новую рубрику (с валидацией уникальности имени).
- `GET /api/categories/:id` — получить одну рубрику со списком её статей.
- `PATCH /api/categories/:id` — частичное обновление полей рубрики.
- `DELETE /api/categories/:id` — удалить рубрику и каскадно стереть её статьи.

### Статьи
- `GET /api/articles?page=1&limit=5&q=текст&categoryId=id` — получить список статей с поиском, фильтром и пагинацией.
- `POST /api/articles` — создать новую статью (увеличивает `articlesCount` у рубрики).
- `GET /api/articles/:id` — детальный просмотр статьи с вложенной рубрикой (`views` увеличивается на 1).
- `PATCH /api/articles/:id` — частичное редактирование полей статьи.
- `DELETE /api/articles/:id` — удалить статью (уменьшает `articlesCount` у рубрики).
